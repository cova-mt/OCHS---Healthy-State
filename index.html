Welcome to Starter 2

This iss a project dedicated to showcasing the accumulated UI work I've done over the past many years. It's meant to be both a reference for use, a living repository for past features, and an experimental space for things I might want to keep around for later.

About this framework:

This framework is intended to be a one-stop-shop starter template for any custom webiste build. It contains a multitude of HTML and Handlebars templates, Sass styles and Javascript for components, and may additional experimental sketches and ideas. The framework is built on Vite, which allows it to function as a bare-bones statc-site generator. Handlebars for Vite is used to provide templating and code-reusability to the Markup. Sass compilation and import is also handled by Vite, as well as JS bundling with Vite's built-in integration of Rollup. The recommended package manager is PNPM, alghough it works just as well on NPM.

Instructions: look over the following sections to understand how each aspect of this framework functions, and what features & components are available to be used. My advice is to start with the minmum you need, then add as you build. This will keep the build light and the file management simple until you require a more complex setup.

Variables

Variables refer to sass variables that are computed during transpilation. This is useful for when you don't need to calculate values in the browser (which may be computationally expensive). They are also necessary for things like media queries, where CSS custom properties aren't allowed. They are also useful for overwriting defaults in 3rd party libraries like Bootstrap.

Vendor Overrides

Most common here are the $fa-font-path variable for FontAwesome and for Bootstrap, $font-family-sans-serif, as well as a number of typography and color theming variables.

Breakpoints

These roughly follow the defaults from Bootstrap, but can be customized to your needs. the range from $xs to $xxl. There is also a matching set of +1 variables that are intended for use in min-width media queries.

Custom Properties (CSS Variables)

These are defined on the :root element and are used throughout the project. This should make the process of theming project fairly straightforward. A simple naming convention is use for the colors, font sizes, and spacing values. The colors are defined as HSB values in order to easily identify a color group by it's hue on a color wheel and judge it's value numerically. This also makes adding additional variables to a color group a bit more intuitive. Another key feature of these variables is that they can be changed by JS, css context, or media query, making them excellent candidates for dynamic values in components that need to change over time, such as --header-height which is re-calculated on page load and resize.

These are sass funcitons that encapsulate styles into a specific context, or add styles based on paramenters to a given selector.
